# Usage Guide

Learn how to use DeepScanBot effectively for your web crawling needs.

## Basic Usage

The simplest way to crawl a website:

```bash
go run ./apps/cli -url https://example.com
```

This will:
- Crawl the start page
- Follow links up to 2 levels deep
- Output results to `crawler_results.txt`

## Common Use Cases

### Single Page Crawl

Crawl only the specified URL without following links:

```bash
go run ./apps/cli -url https://example.com -depth 0
```

### Full Site Crawl

Crawl an entire site with greater depth:

```bash
go run ./apps/cli -url https://example.com -depth 5 -concurrency 20
```

### JSON Output

Export results in JSON format for programmatic processing:

```bash
go run ./apps/cli -url https://example.com -json -output results
```

### Crawl with Proxy

Route traffic through a proxy server:

```bash
go run ./apps/cli -url https://example.com -proxy http://127.0.0.1:8080
```

### Content-Type Filtering

Only download specific file types. **Note:** the value must be quoted as a single argument.

```bash
# Only HTML pages
go run ./apps/cli -url https://example.com -content-types "text/html"

# All content types
go run ./apps/cli -url https://example.com -content-types "*/*"
```

### Unique URLs Only

Avoid processing duplicate URLs:

```bash
go run ./apps/cli -url https://example.com -u
```

### Resume a Crawl

Stop and resume a crawl without recrawling:

```bash
# Initial crawl
go run ./apps/cli -url https://example.com -depth 3 -output my-crawl

# Resume (skips already crawled URLs)
go run ./apps/cli -url https://example.com -depth 3 -resume -output my-crawl
```

### Politely Crawl

Add delays to avoid overwhelming servers:

```bash
go run ./apps/cli -url https://example.com -delay 500ms -host-concurrency 2
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | Error (invalid arguments, crawl failure) |
