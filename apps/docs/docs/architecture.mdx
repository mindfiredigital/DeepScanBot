# Architecture

DeepScanBot follows a modular, package-based architecture designed for extensibility and maintainability.

## Package Structure

### `packages/crawler/` - Core Crawl Engine
The central orchestrator that manages the entire crawling process.

- **crawler.go**: Main crawler logic, URL queuing, concurrency control
- **http.go**: HTTP transport configuration (proxy, TLS, redirects)
- **retry.go**: Retry logic with exponential backoff
- **robots.go**: Robots.txt parsing and compliance checking
- **sitemap.go**: Sitemap.xml discovery and parsing

### `packages/fetcher/` - HTTP Client
Handles HTTP requests with configurable options.

- Configurable timeout and redirect handling
- Proxy support (HTTP/HTTPS)
- TLS verification control
- Content type filtering
- Size limiting
- Retry-After header parsing

### `packages/parser/` - HTML Parser
Extracts links and resources from HTML pages.

- Link extraction (href, src, iframe, form actions)
- Absolute URL resolution
- Supports multiple link sources

### `packages/storage/` - Data Persistence
Manages crawl results and provides multiple output formats.

- In-memory URL tracking with thread safety
- JSON and text output formats
- Resume mode support
- Snapshot isolation for concurrent access

### `packages/logger/` - Logging
Structured logging utility for the crawler.

- Configurable log levels
- Formatted output

### `packages/types/` - Shared Types
Common type definitions used across all packages.

- `URLEntry`, `CrawlReport`, `CrawlSummary`
- `FetchResult`, `CrawlerOptions`

## Data Flow

1. **CLI Parsing**: `apps/cli/main.go` parses command-line flags
2. **Crawler Init**: Creates a `Crawler` instance with options
3. **URL Enqueuing**: Start URL is added to the crawl queue
4. **Concurrent Crawling**: Workers fetch URLs concurrently with host-level rate limiting
5. **Robots Check**: Each URL is checked against robots.txt
6. **HTTP Fetching**: URLs are fetched via the fetcher package
7. **Link Parsing**: HTML responses are parsed for links
8. **Result Storage**: Results are stored in-memory with thread safety
9. **Report Generation**: Crawl report is generated on completion
10. **Output Writing**: Results are written to file (JSON or text)

## Concurrency Model

- **Global Semaphore**: Limits total concurrent requests
- **Per-Host Semaphore**: Limits concurrent requests per host
- **Crawl Delay**: Configurable politeness delay between host requests
- **WaitGroup**: Tracks active crawl goroutines
